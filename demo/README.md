# AFINs Demo

Start with the notebooks in order:

- `00_one_shot_afin.ipynb`: one-shot posteriors from Gaussian and flow AFINs.
- `01_afin_as_proposal.ipynb`: AFINs as SNIS or independence-MH proposals.
- `02_extending_afin.ipynb`: how to extend task families and training
  distributions for new research.
- `03_real_world_logistic.ipynb`: one OpenML binary logistic-regression task
  with the standard prior.
- `04_standard_prior_openml.ipynb`: the fixed Flow + SNIS OpenML
  posterior-predictive benchmark table.

Notebook 03 is intentionally a single-task walkthrough, while notebook 04 is
the cached multi-dataset benchmark-table baseline.

The notebooks download pretrained AFINs from
[`joohwanko/AFINs`](https://huggingface.co/joohwanko/AFINs).
Notebook-only plumbing such as timing, tables, and coefficient plots lives in
`src/afin/demo.py`. The heavier OpenML loop behind notebook 04 lives in
`benchmarks/compare_tabpfn_openml.py`, with cached results in
`demo/assets/openml_standard_prior_benchmark.csv`.

The demo API is intentionally small:

```python
from afin import AFIN, GaussianPrior, LinearGaussian, BernoulliLogit, LinearStudentT, infer

prior = GaussianPrior(0, 1)
observed = [
    LinearGaussian(design_matrix=X, sigma=0.35).observe(y),
    BernoulliLogit(design_matrix=X_binary).observe(y_binary),
    LinearStudentT(design_matrix=X_tail, sigma=0.60, df=4).observe(y_tail),
]
posterior = infer(AFIN("flow"), prior, observed, num_samples=4000, seed=0)
```

`infer(...)` converts this factor declaration into the typed task consumed by
an AFIN, prints a short start/finish line, and stores timing on
`posterior.runtime_seconds`. For AFINs that value is the timed neural posterior
forward pass; the full wall time is kept as `posterior.total_runtime_seconds`.
Set `log_prob=True` when you want log-density values attached to the returned
posterior. The homogeneous Gaussian example uses its exact conjugate posterior as
ground truth. For the mixed example, `REFERENCE_PLOTS` controls whether figures
use the unnormalized target grid, a NUTS sample reference, or both; the default
uses NUTS and prints metrics for both Gaussian and flow posterior families.

Static README/demo figure assets are generated by the scripts we keep in this
folder:

```bash
uv run python demo/make_one_shot_figure.py
uv run python demo/make_openml_comparison_figure.py
uv run python demo/make_sampler_gif.py
```
